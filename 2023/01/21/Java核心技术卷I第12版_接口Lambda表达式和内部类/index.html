Java核心技术卷I第12版_接口Lambda表达式和内部类

一、接口

在java中，接口不是一个类（尽管用起来感觉和类一样），而是对某些类的一组需求。当一个类声明为某个接口的实现，并且类中定义了接口中的全部方法，那么我们就说这个类实现了一个接口。

接口中的方法都自动默认为public的，因此可以在方法声明前省略关键字public（一般采用省略方式避免冗余）。在java8中允许接口中存在static的方法，尽管没有理由认为这是不合法的，但是这似乎违背了将接口作为抽象规范的初衷。在java9中允许接口中存在private的方法，但是由于private方法只能在接口内部使用，所以用途很有限，一般只是作为接口中其他方法的辅助方法而已。

接口没有实例字段，但是接口中可以包含常量。接口不是类，不能new interface实例化，因此接口中的属性都默认是常量，默认是public static final的（一般采用省略方式避免冗余），但是接口变量可以引用一个实现了该接口的类。

接口间也可以继承（代码复用），语法同类继承一样。

为什么不将接口设计为抽象基类？因为java中的类只能单一继承不能多重继承，所以当接口设计为抽象基类时，一个类最多只能实现一个接口。而实际应用中我们常常需要一个实现了多个接口的类，因此不能将接口设计为抽象基类。

可以在接口定义中，用default关键字声明一个方法的默认实现。这样是为了”接口演化”，例如在java8之前Collection接口不存在stream方法，java8中为Collection接口新增了stream方法，那么之前的代码中实现了Collection接口的类在java8环境下不能通过编译。使用default关键字可以消除这个弊端。

默认方法冲突的问题：
  • 基类与接口默认方法冲突：类优先原则，接口默认方法被忽略。
  • 两个接口默认方法冲突（至少其中一个接口中提供了默认方法）：程序员决定二义性问题。

此外，java的lambda表达式还有一些注意事项。

  • Lambda表达式允许使用更简洁的代码来创建只有一个抽象方法的接口（这种接口被称为函数式接口）的实例。

  • Lambda表达式的类型，也被称为“目标类型”，Lambda表达式的__目标类型必须是“函数式接口”__。由于Lambda表达式的结果就是被当成对象/实例，因此程序中完全可以使用Lambda表达式进行赋值，赋值给相应的函数式接口对象。（注意是函数式接口Comparator，不是实现了函数式接口的类Cmp1）

Java核心技术卷I第12版_Java编程环境

JRE

Java Runtime Environment，Java运行环境，包括Java虚拟机（jvm）、Java核心类库和支持文件。由于JRE并不包含开发环境，所以没有包含任何开发工具，如编译器、调试器和其它工具。

当你只需要运行已经编译好的Java程序时，而并不需要写Java代码、编译Java代码或者调试Java代码等等时可以只安装JRE。

JDK

Java Development Kit，Java开发工具包，包含java开发工具以及该jdk专用的jre。因为java开发工具也是由java语言编写的，因此要使用java开发工具必然要具备java运行环境。

JDK自带的jre称为专用jre，单独安装的jre称为公共jre。如果安装了JDK的话，其实是没必要再安装公共jre的。因为公共JRE是一个独立的JRE系统，会单独安装在系统的其它路径下。公共JRE会向Internet Explorer浏览器和系统中注册Java运行时环境。通过这种方式，系统中任何应用程序都可以使用公用JRE。由于现在网页上执行Applet的机会越来越少，而且完成可以选择JDK目录下的JRE来运行任何Java程序，因此没有太大必要安装公共JRE。只要正常安装JDK，指定好环境变量后，就OK了

注：Applet是采用Java编程语言编写的小应用程序，该程序可以包含在 HTML（标准通用标记语言的一个应用）页中，与在页中包含图像的方式大致相同。

Java三大版本：Java SE、Java EE、Java ME

JSE即标准版（standard edition），包含了Java核心类库，主要用来开发桌面应用。

EE即企业版（enterprise edition），包含SE，又有扩展部分（Servlet，JDBC等），主要用来开发分布式网络程序。

ME即微型版（micro edition），包含了SE中部分类库，又有自己扩展部分，主要用来做移动类、嵌入式开发。

Java Web

定义：Java Web应用由一组Servlet、HTML页、类、以及其它可以被绑定的资源构成。它可以在各种供应商提供的实现Servlet规范的 Servlet容器 中运行。

重点在于Web，显然Java Web是企业级开发的重要组成部分，只能算是企业级Java开发的一个子集，主要包括jsp、servlet等为基础的一系列技术，当然现在的企业开发已经见不到jsp和servlet了，取而代之的是SSM或Spring Boot等一系列开发框架（但其实这些框架不仅仅局限于Web开发）

对于现在的开发而已，这已经被分为两个领域了，分为“服务端”和“客户端”，也就是两个岗位，一个是后端，一个是前端。

JSP

JSP全称Java Server Pages，是一种动态网页开发技术。它使用JSP标签在HTML网页中插入Java代码。标签通常以<%开头以%>结束。

JSP是一种Java servlet，主要用于实现Java web应用程序的用户界面部分。网页开发者们通过结合HTML代码、XHTML代码、XML元素以及嵌入JSP操作和命令来编写JSP。

JSP通过网页表单获取用户输入数据、访问数据库及其他数据源，然后动态地创建网页。

JSP标签有多种功能，比如访问数据库、记录用户选择信息、访问JavaBeans组件等，还可以在不同的网页中传递控制信息和共享信息。

3、验证是否安装成功：javac –version

注：一般来说，安装时会自动添加相关的环境变量，如果安装时没有报错，但执行命令却提示未找到该命令，这时应该先检查环境变量（Java/jdk-x.x.x/bin）是否被添加。

4、如果需要查看java源码，则要安装源文件和文档

4.1 进入安装根目录下的lib目录找到src.zip文件

注：src.zip文件包含了所有公共类库的源代码，也就是说我们在写java程序用到的 java.lang.String ,java.lang.date等所有的类都能在src压缩包中找到源代码。

4.2 将src.zip文件解压缩

一般我们在src.zip的同级目录创建src目录，然后将src.zip文件解压缩到src目录下。

注：在集成开发环境下，一般我们不需要自己去解压缩src.zip文件，只需要在集成开发环境中设置源代码路径(就是src.zip的路径)即可。

5、第一个java程序：HelloWorld

简单来说整个流程就是：javac将.java源代码文件编译成.class文件，然后jvm加载.class文件运行程序。

注：运行时不要加.class后缀。用java Helloworld.class是错误的！另外，class文件是根据类名生成的，并不是根据java文件名，因此，一个java源代码文件在定义了多个类的情况下会生成对应的多个class文件！

6、class文件中的字节码

HelloWorld.class文件中前4个字节是固定的magic number: cafe babe，图中是符合预期的。javac将java代码编译为字节码后，jvm将字节码再转换为机器指令运行。因此可见，字节码是中间代码，而通过jvm可以将相同的字节码在不同机器上转为不同的机器指令，从而实现跨平台的功能。

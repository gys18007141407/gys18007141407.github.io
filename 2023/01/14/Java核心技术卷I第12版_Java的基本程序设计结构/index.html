Java核心技术卷I第12版_Java的基本程序设计结构

一、main方法

  • 每个java程序都包含了一个main方法，main方法必须声明为public static类型，并且返回值是void类型。

与c++不同，java是纯面向对象的，因此不存在独立于类的函数。main方法一定是由一个外壳类包装起来的。

main方法并不依托于实例来调用，因此要声明为static类型。main方法是static类型的，那么方法体中也只能调用static方法，调用非static的方法要先实例化对象，利用对象调用其拥有的方法。

java解释器强制要求main方法的访问权限是public的。

main方法的返回值必须是void。c/c++的main函数返回值会被_exit系统调用接收，之后送给父进程（比如shell），让父进程知道他的这个子进程的退出状态（父进程要用wait/waitpid来获取子进程的退出状态信息，然后回收资源防止子进程成为僵尸进程）。c/c++没有jvm，因此只能在main函数中显式return来将自己的退出码告诉父进程，但有些编译器也会在编译过程中帮忙，这时候将main函数返回值声明为void类型也是可以的。在java中，这个退出过程是由jvm进行控制的，在发生以下两种情况时，java程序会终止其所有行为并退出:所有不是后台守护线程的线程全部终止（jvm在启动时创建了一系列守护线程运行在后台处理任务，比如垃圾回收等）;某个线程调用了Runtime类或者System类的exit方法，并且安全管理器并不禁止exit操作。上面的两种情况中，第一种情况就是正常退出，jvm设置返回码为0。第二种情况一旦发生，jvm不会管main方法是否执行完毕，他都会终止所有行为并退出，这时候main方法的返回值是没有任何意义的，因为exit方法的参数已经提供了返回码。综上，在java中main方法的返回值就被固定为void。

二、数据类型

  • 8种基本类型：4种整型（byte、short、int、long），2种浮点型（float、double）、1种字符型（char)、1种布尔型（boolean）

1、整型

我们知道c/c++中int占的字节数和平台有关，long实际上是long int类型就等于int，8字节的整型使用long long类型，一定要精准使用特定字节长度整型的话需要使用int8、int16、int32、int64。 与c/c++不同，java的int固定为4字节，long为8字节。

在 Java SE 7 和更高版本中,数字中数字之间的任何位置都可以出现任何数量的下划线字符，可以用来分隔数字中的数字以提高代码的可读性。如二进制数0b1001_0010_1101。

在c/c++中可以定义unsigned int、unsigned long long等无符号整型类型。但是，java中没有类似于unsigned这样的关键字，只提供了toUnsignedInt(byte)、toUnsignedLong(int)、toUnsignedString()这一系列方法来完成无符号数的转换。

2、浮点型

与c/c++一样，在java中float为单精度浮点数，double为双精度浮点数，double类型的有效数字位数是float类型的两倍。

在未提前声明的类型自动推导时，不在小数后面加f则默认使用double类型。

3、char型

主要注意一点Unicode编码，如\u03c0是Unicode编码的希腊字母π，\u0022是Unicode编码的双引号”等等。Unicode转义的字符会在编译器解析代码之前处理，所以下面的示例等价于System.out.println(""+"")，打印的就是空字符串。针对于这样的机制有可能会在程序中发生诸如语法错误等意料之外的事情。（**任何时候都要注意\u**，例如路径字符串中使用了"C:\user\xxx"，此时u后面没有跟着4个16进制数，处理时Unicode编码字符时就发生了语法错误）

为什么需要Unicode编码？或者说Unicode编码的目的是什么？因为在Unicode之前已经出现了很多种编码标准，如美国的ASCII、中国的GB 18030等等，这样就出现了两个问题：一个特定代码值在不同的编码标准下可能对应了不同的字符；单字节编码和多字节编码导致的编码长度不同。所以Unicode就是为了统一标准而诞生的新编码标准。

字符的定义：各种文字和符号的总称，包括各国家文字、标点符号、图形符号、数字等。 也就是说，它是一个信息单位，一个数字是一个字符，一个文字是一个字符，一个标点符号也是一个字符。

字符集的定义：字符的集合就叫字符集。不同集合支持的字符范围自然也不一样，譬如ASCII只支持英文，GB18030支持中文等等。在字符集中，有一个码表的存在，每一个字符在各自的字符集中对应着一个唯一的码。但是同一个字符在不同字符集中的码是不一样的，譬如字符“中”在Unicode和GB18030中就分别对应着不同的码(20013与54992)。

字符编码的定义：定义字符集中的字符如何编码为特定的二进制数，以便在计算机中存储（就是将字符在字符集中的对应位置化为二进制）。字符集与字符编码的一个例外就是Unicode字符集，它有多种编码实现(UTF-8,UTF-16,UTF-32等)

5、常量

java中使用final关键字定义常量，与c/c++不一样，java中的const是保留字，目前还没有使用。

6、枚举类型

和c/c++几乎一致，使用enum关键字像定义结构体一样定义一个枚举类型即可。

7、字符串

在java中，一个String对象和一个非String对象进行拼接时，非String对象会被转成String对象（任何一个java对象都能转换成String对象）。

在java中，String类没有提供任何方法来修改String的某个字符，因此java中的String类可以看作是字符串常量！要想完成修改目的则必须使用拼接操作来获得，修改后的字符串与原来的字符串的底层地址是不一样的。（jvm垃圾回收会释放原String的内存）

java的String类似于c的const char *类型，null串和空串是不一样的（类似空指针和指针指向长度为0的字符串的区别），可以用如下条件在java中判断字符串不是null也不是空串。

与c/c++不同，在ava中比较两个String是否相等要用equals方法。c用strcmp比较，c++因为string类重载了==运算符所以可以使用==判断，而java中如果使用==比较String对象的话就优点类似于指针比较了，这在绝大部分场景下都是不符合预期结果的。

如果要多次拼接短字符串来合成目标字符串的话，String类的拼接操作既耗时间又耗空间。这时最好是使用StringBuilder类来达到我们的目的。另外有一点要知道的是，StringBuffer类的效率不如StringBuilder类，但StringBuffer类允许多线程添加或删除字符（但通常我们是在单线程下操作一个字符串），这两个类的api是一样的。
